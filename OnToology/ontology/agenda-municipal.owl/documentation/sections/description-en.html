Vocabulary for the representation of data on the events that take place within the municipal agenda and their participants: Description back to ToC

Figure 1 presents the classes and properties of the vocabulary for the representation of data on the events that take place within the municipal agenda and their participants
The main concept of this vocabulary is [esagm:Event](#Evento) for which the classes [schema:Event](http://schema.org/Event) and [schema:Place](http://schema.org/Place) of the schema.org vocabulary are reused. A Place can be a municipal building or center [esequip:Equipamiento](http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/equipamiento-municipal/index-en.html) such as the town hall, a cultural center, etc. A place has an address that may be normalized using the properties of the class [escjr:DireccionPostal](http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/direccion-postal#DireccionPostal) and the classes [escjr:Via](http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/callejero#Via) and [escjr:Portal](http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/callejero#Portal). The full address can be represented with the property [locn:fullAddress](http://www.w3.org/ns/locn#fullAddress). An event may have documentation [schema:DigitalDocument](http://schema.org/DigitalDocument) such as the Agenda or the Minutes of the meeting. A member of an event can have the role [esagm:rol](#rol) of organizer, chair or attendee. The events include the participation of several [foaf:Agent](http://xmlns.com/foaf/spec/#term_Agent); it is worth mentioning that the set of agents includes people, organizations such as institutions or companies and groups. The positions of the participants in an event and its administrative units are represented through the classes [org:Role](https://www.w3.org/TR/vocab-org/#org:Role), [org:Organization](https://www.w3.org/TR/vocab-org/#class-organization) and [org:Membership](https://www.w3.org/TR/vocab-org/#class-membership). An organization has an area of ​​competence that is a governmental administrative region represented by the class [dbo:GovernmentalAministrativeRegion](http://dbpedia.org/ontology/GovernmentalAdministrativeRegion) and it can be a district [esadm:Distrito](http://vocab.linkeddata.es/datosabiertos/def/sector-publico/territorio#Distrito) or a municipality [esadm:Municipio](http://vocab.linkeddata.es/datosabiertos/def/sector-publico/territorio#Municipio). Additionally, in this vocabulary several SKOS concept schemes have been used to represent the type of access, type of session, type of event or channel of an Event. In Figure 1 you can see several relationships with the SKOS concept class, however, the specific values ​​have not been specified because they are different depending on the taxonomy they represent. For example, the type of session property [esagm:tipoSesion](#tipoSesion) represents whether the session is Ordinary, Extraordinary or Extraordinary Urgent.

3.1 Examples of vocabulary usebackindex

In order to facilitate the understanding of some classes and properties of this vocabulary, some examples will be provided below. First, for the representation of an event, an instance of the class [esagm:Evento](#Evento) is created with the properties [schema:name](http://schema.org/name) (title of the event), [schema:startDate](https://schema.org/startDate) and [esagm:tipoEvento](#tipoEvento) according to the taxonomy of event types that has been defined, in the example the event belongs to the type internal meeting. A unique identifier of the event has been generated from the combination of its title and its start date and time. The event is associated through the property [schema:location](https://schema.org/location) with an instance of [schema:Place](https://schema.org/Place) where the event takes place, and the place in turn has a [schema:name](https://schema.org/name) and a [schema:address](https://schema.org/address). The address is a [esdir:DireccionPostal](http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/direccion-postal#DireccionPostal) that contains the normalized address as well as the [locn:fullAddress](http://www.w3.org/ns/locn#fullAddress). The event has as territorial scope [esagm:ambitoEvento](#ambitoEvento), the Madrid municipality that has been defined by the National Geographic Institute (IGN) as the municipality-type resource with code 28079. In addition, an instance of the class [esagm:RolIntegranteEvento](#RolIntegranteEvento) is defined, that allows to relate the event to each chair or attendee, in this case information is provided about the chair of the event for which a unique identifier has been generated from its names and surnames.
